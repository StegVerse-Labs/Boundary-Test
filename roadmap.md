# Boundary Test Roadmap

## Purpose

This roadmap tracks planned evolution of the Boundary Test repository.

All roadmap items must preserve the review-only posture of the repository.

## v0.1.x Stabilization

- [ ] Verify all documented files exist and are referenced from the repository index.
- [ ] Verify CI passes all validation runners.
- [ ] Verify REVIEW-ONLY remains the top-level outcome.
- [ ] Verify invalid-control fixtures continue to fail.

## v0.2.0 External Review Package

- [ ] Add reviewer evidence packet.
- [ ] Add fixture export bundle.
- [ ] Add machine-readable test result schema.
- [ ] Add comparison report generator.

## v0.3.0 Multi-Framework Boundary Testing

- [ ] Support multiple adjacent governance artifacts.
- [ ] Add framework-neutral comparison fixtures.
- [ ] Add boundary drift detection examples.
- [ ] Add cross-framework non-claim validation.

## Long-Term Goal

Enable independent reviewers to evaluate governance-boundary behavior without creating conformance, interoperability, delegation, execution authority, or consequence authority claims.
